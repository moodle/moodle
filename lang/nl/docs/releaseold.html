Oude Release notes

Moodle 1.4.5 (7 mei 2005)

(Omdat deze release belangrijke beveiligingsproblemen oplost, raden we aan de vorige Moodle-versie zo snel mogelijk te upgraden.)

Enkele belangrijke beveiligingsproblemen opgelost
  • Voorzorgsmaatregelen tegen aanvallen op beheersfuncties.
  • Les: Veiliger verbergen van lessen (#2733)
Enkele nieuwigheden
  • Flash Media Player: Het afspelen kan nu beginnen voor het bestand volledig is gedownload.
  • Toegevoegde en geüpdate ondersteuning voor Duits en Turks.
  • Release notes verwijzen nu naar de bijhorende fouten.
  • Voorlaad aanvragen van Google Web Accelerator, Mozilla enz worden verhinderd.
Performantieverbeteringen
  • Optie toegevoegd om de taallijst te cachen. Dit bespaart ongeveer 10MB geheugen en 40 includes. Bekijk config-dist.php voor de details over hoe je dit moet inschakelen.
  • Belangrijke verbeteringen naar eficiëntie voor Postgres databases bij invoeren. Ook heel wat aanroepen naar insert_record() die het record id niet nodig hebben aangepast met het niet meer vragen naar het record id.
Andere herstelde fouten
  • HTML Editor: problemen met sommige karaktercoderingen opgelost.
  • Test: fouten opgelost aan export/import menu (#2963)
  • Test: geef geen fatale fout bij een onbekend vraagtype, maar probeer verder te gaan.
  • Cursus Sociaal Forum: misleidende link naar lidmaatschap opzeggen verwijderd (#2945).
  • Bijkomende / aangepaste vertalingen.
  • Forums en Groepen: Laat leerlingen die niet in een groep zitten toe te posten in een forum dat niet in groepen verdeeld is, zelfs al is de cursus zelf wel in groepen verdeeld.
  • HTML Editor instellingen voor de site: Hersteld Kan de wijzigingen in de configuratie van de editor niet bewaren formulierfout.
  • Cron: Enkele waarschuwingen verwijderd.
  • Keuze: Keuze toont een fout aantal antwoorden (#2452).
  • Betere ondersteuning voor PHP5: Upload van bestanden gebruikt nu $_FILES, eerder dan de niet meer ondersteunde $HTTP_POST_FILES (#2828)
  • Backup/Restore: Manuele links van lessen naar bronnen worden nu overal behouden tussen een backup en een restore. (zie deze discussie)
  • Naamgeving voor Postgres hersteld.
  • I18N support: Codering tijdens login juist zetten (#2770).
  • I18N support: Sommige verknoeide karakters hersteld bij het gebruiken van utf-8 in HTML Editor (#2763)
  • Flash Media Player: Automatisch laden van geluidsbestanden uitschakelen.
  • Flash Media Player: Betere afhandeling van bestandsnamen.
  • Postgres: Sequences uit sequence hersteld(#2742).
  • Backup: Memory limit overrun hersteld in de backup configuratiepagina.
  • Forums: Foutwaarschuwingen hersteld wanneer een leraar in een forum post (#2723).

 

Moodle 1.4.4 (7th Maart, 2005)

Belangrijke herstellingen aan de beveiliging
  • Sterke validatiefuncties voor URL's toegevoegd.
  • Voorzorgsmaatregels tegen tampering van instellingen.
  • Tighter input validation in Quiz, HTML editor and course edit.
  • Meer controle over de taalinstellingen van gebruikers.
  • Veiliger opkuisen van invoer met HTML-opmaak.
Enkele nieuwigheden
  • NNieuwe MP3 filter met Flashgebaseerde mp3-speler.

Performantieverbeteringen
  • Belangrijke performantieverbeteringen in de forumcode.
  • Verschillende verbeteringen door het verwijderen van onnodige vragen en sorteringen in de databank.
  • Belangrijke verbeteringen in het behandelen van sessies. Het downloaden van grote bestanden of het bekijken van de live logs zal niet langer het navigeren door andere vensters vertragen.

  • Installer: Vele verbeteringen uit HEAD (inbegrepen #2625) .

  • Enkele zoekproblemen in forums hersteld.
  • Sorteren van fouten in forums hersteld(#2613).
  • Problemen met het zoeken naar gebruikers met spaties achteraan opgelost.
  • Enkele problemen met hardgecodeerd /admin pad hersteld.
  • Enkele groepgerelateerde fouten hersteld(#2133, #2550, #2363)
  • Geen HTML digests sturen naar mensen die platte tekst mail willen.
  • SCORM module — fout in navigatiebalk hersteld.
  • Chat: Maximum socket read size verhoogd om lege pagina's en herhaalde boodschappen te voorkomen.
  • Bestanden: "Parent Folder" icoon toegevoegd, (#2491).
  • Postgres ondersteuning: consistentie hersteld in mdl_log table.
  • Authenticatie plugins: Beter opkuisen en inkorten van gegevens.
  • Authenticatie plugins: Extern authenticatie probleem opgelost — wanneer aangemeld als gast, inloggenals een extern geauthenticeerde gebruiker faalt als het de eerste keer is dat die account aanmeld.
  • Authenticatie LDAP: Een teruggekeerde fout uit 1.4.2 hersteld waarbij een puntkomma op het einde van sommige velden de authenticatie verhinderd.
  • Authenticatie DB: Databank Authenticatie haalt de gebruikersgegevens niet correct op (#2319).
  • Paypal aanmeldingsplugin: Nu kan tekst die naar PayPal gestuurd wordt, opgekuist worden om sommige diacritische tekens te vermijden (tot nu voor Spaans) die het aanmeldingsproces kunnen doen mislopen. Enkel Duitse en Franse diacritische tekens worden nu door Paypal ondersteund. Deze functie kan gemakkelijk uitgebreid worden naar andere tekens.
  • Backup/restore: Als de gebruiker geen beheerder is en de teruggezette cursus bevat geen leraren, voeg dan een leraar toe (#2381)
  • Backup/restore: Fouten hersteld in cursussen met apostrophes in de volledig naam.
  • Backup/restore: Nu worden groepleden enkel toegevoegd als we gekozen hebben om gebruikers mee in te voegen (#2486).
  • Backup/Restore: Algemene opkuis.
  • Glossary: Nu worden 0 waarderingen ondersteund in de woordenlijst (#2582) en beter PHP5 ondersteuning (#2581).
  • Woordenlijst & Blackboard: herstellingen aan import (#2310).
  • Kalender: robuuster afhandeling van foute gegevens.
  • Kalender: HTML hersteld (#2350).
  • Sorteren en volgorde van cursussen was stuk in 1.4.3 en kon soms verhinderen dat cursussen aangemaakt werden — alle bekende problemen met sorteren zijn nu hersteld, inclusief sommige get_categories() calls die soms verschijnen in "Mijn cursussen" lijst.
  • Wiki: Verschillende herstelde fouten.
  • Loglive heeft nu een geschikte voettekst.
  • Algemene opkuis van de code.

  • Language can be specified for the current session (see menu on home page and login page) and can be changed on any page by something to the url, eg: http://moodle.org/?lang=ru
  • Language menu can be restricted to a subset of languages and even removed.
  • Truetype fonts are now part of the language packs
  • ALL dates/times can now be reformatted as part of the language pack
  • A small but annoying bug was fixed with the web-based language editor on Windows servers that caused language files to grow
Developers
  • Global debug variable gives more feedback about uninitialised variables and the like
  • Various fixes to tidy up and further standardise some of the PHP code
  • Various fixes for robustness (ensuring variables are always initialised)
Admin
  • Some changed options and code in config-dist.php (if upgrading, compare with your current config.php or rebuild your config.php from scratch)
  • Many changes to themes - authors of custom themes should read theme/UPGRADE.txt
  • New moodle.org registration function for security notifications etc
  • Admin directory can be moved from /admin to something else (see config-dist.php)
  • User listing is now paged, and can now be searched for strings
  • Module management: activity modules can now be individually disabled or deleted completely
  • Improvements to the language editing page
  • Better checking of PHP environment during installation, with warnings
  • Cleaner installation procedure.
User management
  • New role: Course creator - just like teacher but can also create new courses
  • New authentication plug-in named "manual", prevents users from creating their own accounts
  • "Forgot Password" routine now uses an email confirmation before resetting password
Courses
  • Activities on the course page now have a little button to hide/show that activity to students
  • Whole topics/weeks can also now be hidden or shown
  • The list of participants now has some paging, to cope with very large classes
  • Zip/Unzip is now supported by an internal zip library, removing the dependence on external programs to do this. This also means that zip/unzip now works on Windows.
Assignments
  • Assignment submissions can be sorted
  • Assignment notification mail isn't sent to unenrolled students
Forums
  • Display of forum discussions list is greatly improved
  • Whole discussions can be moved to any other forum with a single click
  • Date/time of posts is now "last modified", not "created"
  • Blank subject-lines or messages are now prevented
  • More intelligent parsing and display of text in posts
  • Force Subscribe now works on the "Site News" forum (mails all active students/teachers)
  • Better handling of mailouts to prevent the possibility of double copies that sometimes happened with very large classes
Workshop Assignment module (not included: available soon as a separate download)
  • A very interesting new module contributed by Ray Kingdon,
  • A flexible tool that implements a variety of peer-grading and example-setting schemes that involve a whole group in examining and grading pieces of work.
Quizzes
  • New question type, Random Question (selects from all questions in the same category)
  • New question type, Matching Question (student must match answers to questions)
  • New question type, Random Matching Short-Answers (randomly creates Matching Questions from the available Short Answer questions in the same category)
  • Quiz question import from text files (modular design allows expansion to new formats - a beta Blackboard module is included)
  • 'Create multiple questions' wizard to quickly create a random quiz
  • Quiz questions may be randomly shuffled
  • Quiz answers (multiple choice, etc) may be randomly shuffled
  • Fixed off-by-one error when "Save Grades" button wasn't used.
  • Regrading works properly if the quiz has been changed
  • Editing a question after it's already been used in a quiz no longer affects quiz results
  • Multiple choice answers are labelled a,b,c not 1,2,3
Resources
  • Better performance when editing large text or HTML resources
  • Plain images are now display centered within a proper web page (not raw).
  • Wiki format added as a new option for Resource pages
Custom Themes
  • A number of new styles have been added to all built-in themes. Some of these are ESSENTIAL to viewing some of the new features. PLEASE SEE theme/UPGRADE.txt TO SEE THE CHANGES YOU WILL HAVE TO MAKE TO OLDER THEMES.
  • Custom themes now have the capability to re-define all the little button graphics. There is a new entry in theme/x/config.php ($THEME->custompix) and a new optional subdirectory "pix". See the theme cordoroyblue for an example.

Older releases can be seen in the Moodle.org announcement forum.
