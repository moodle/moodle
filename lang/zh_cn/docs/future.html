展望

随着Moodle的逐渐成熟，它的发展方向被社区里的开发者和用户影响得越来越多。一个用来存储对新特性的请求和它们的状态的动态数据库在moodle.org/bugs。无论创意、代码、反馈还是鼓励，任何形式的贡献都是欢迎的。看开发者手册和社区论坛可以得到更详细的信息。您还可以通过付费来加快某个您迫切需要的特性的实现，moodle.com/development有详细信息和报价。

以下是未来发展的路标，不过它总在赞助商和开发者的影响下而改变。

1.6版

Blog做为思考活动的主要工具(提供用户、课程和站点种子)

数据库模块，一个用来关联数据项、搜索和浏览的通用工具

以活动或课程格式的方式和LAMS集成

更好的统计功能

处理接收到的邮件

基本的Web Services API

1.7版

我的Moodle页提供对所有课程的可定制的纵览，包括RSS种子等

改进用户资料页，在一个半开放的主页上集成Blog、种子等

初步支持IMS LD Level A，允许导入和导出

全面支持SCORM 2004

改进Web Services API

和更多的容器集成

2.0版

有条件的活动，允许相关或强制路径

全面支持IMS LD标准

改进小组模式，允许在站点和活动级定义组

改进角色的实现，允许在站点、课程和活动级自定义角色

使用HTML模板自定义主页

随着Moodle在它一直关注的协作方面的自然进化，一些令人激动的发展将使它更加具有网络意识（Network-aware）。以后还会有更多特性写在这里。
